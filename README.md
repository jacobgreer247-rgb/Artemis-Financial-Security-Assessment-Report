# Artemis-Financial-Security-Assessment-Report

Briefly summarize your client, Artemis Financial, and its software requirements. Who was the client? What issue did the company want you to address?

Artemis Financial was a company that needed help improving the security of their software. They wanted to make sure sensitive financial data was protected and that their system did not have vulnerabilities that could be exploited. The main issue was identifying and fixing security risks in their application.

What did you do well when you found your client’s software security vulnerabilities? Why is it important to code securely? What value does software security add to a company’s overall well-being?

I think I did well at identifying vulnerabilities and understanding where the risks were in the software. Secure coding is important because it protects user data and prevents attacks. If software is not secure, it can lead to data leaks, financial loss, and damage to a company’s reputation.

Which part of the vulnerability assessment was challenging or helpful to you?
One challenging part was understanding all the vulnerabilities and figuring out which ones were more important to fix. It was helpful to use tools and reports because they made it easier to see where the issues were and what needed attention.

How did you increase layers of security? In the future, what would you use to assess vulnerabilities and decide which mitigation techniques to use?

I increased security by fixing vulnerabilities, using better coding practices, and adding things like encryption and input validation. In the future, I would use tools like dependency check and follow secure coding guidelines to find and fix issues more efficiently.

How did you make certain the code and software application were functional and secure? After refactoring the code, how did you check to see whether you introduced new vulnerabilities?

I made sure the code still worked by testing it after making changes and checking that everything ran correctly. I also rechecked for vulnerabilities to make sure no new issues were introduced after refactoring.

What resources, tools, or coding practices did you use that might be helpful in future assignments or tasks?

I used tools like dependency check and followed secure coding practices. I also used documentation and testing to help understand and fix vulnerabilities.

Employers sometimes ask for examples of work that you have successfully completed to show your skills, knowledge, and experience. What might you show future employers from this assignment?

I would show my vulnerability assessment or secure software report to demonstrate that I understand how to identify security risks and improve software security. It shows that I can think about both functionality and protection at the same time.
